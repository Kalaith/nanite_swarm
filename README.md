# Nanite Swarm

Nanite Swarm is a 2D grid-based automation and logistics game built with Rust + Macroquad. This README focuses on what is currently implemented in this repo, plus how to run, generate assets, and navigate the code.

## Why Play

- **Spatial logistics puzzles that fight back**: power grids, conduits, dust buildup, and collapse cascades create real pressure.
- **Permanent terrain tradeoffs**: harvesting mountains/forests leaves irreversible scars and long-term penalties.
- **Visible machine evolution**: the core mutates through micro?stages as you progress.
- **Automation that breathes**: drones physically move, idle, fail, and recover on screen.

## Current State (Implemented)

### Core Gameplay Loop
- Land on a single planet with a revealed core area
- Place buildings on a grid to harvest resources and expand
- Drones carry minerals from drills to the core
- Power grid determines whether buildings operate
- Research unlocks new buildings and upgrades

### Pressure + Failure Systems
- Dust accumulation affects efficiency, drone speed, power leakage, and can stall buildings
- Power collapse cascade triggers after sustained negative power (recoverable but painful)

### Terrain + Harvesting
- Terrain types: Ground, Mountain, Forest, Water, Rough, Void
- Mountains and forests can be harvested into new terrain states
- Harvesting grants minerals or biomass
- Water/Void are non-buildable (bridges allow conduit crossings)
- **Permanent consequences:**
  - Mountain harvest permanently removes turbine bonus
  - Forest harvest increases dust/pollution locally

### Buildings
- **Core**: central hub, provides power
- **Drill**: produces minerals, spawns drones
- **Conduit**: connects buildings and transmits power
- **Bridge**: enables conduits to cross Water/Void
- **Power Node**: extends power range
- **Wind Turbine**: generates power (bonus on mountains)
- **Server Bank**: consumes power to generate data
- **Sweeper**: clears dust in a radius

### Logistics + Power Rules
- Conduits cannot overlap other buildings
- Bridges allow conduits across Water/Void tiles
- Conduit placement supports drag-to-paint with pathing
- Power spreads from the Core through conduits and nodes
- Buildings show powered/unpowered states and UI feedback

### Research System
- Visual research tree with prerequisites
- Data generated by server banks
- Unlocks new tech and buildings

### Directives (Short-Term Goals)
- 10?minute directive cycle with rotating objectives and data rewards
- Examples: sustain +power, operate drills, run server banks, harvest forests

### Offline Progression
- Simulated offline time with a 4-hour ?battery? at full speed
- Hibernation mode at reduced speed after the battery drains
- UI banner shows offline simulation results

### UI + Screens
- Main menu, settings, planetary view, research view, interplanetary view
- HUD displays resources, power balance, and time remaining
- Build palette and right-side intel panel
- Help overlay and hotkeys

### Procedural Art Pipeline
- 2D tile generator for terrain + buildings
- Conduit variants (corners/tees/cross/straight) are generated
- Core has multiple early-stage visuals
- Game loads textures at runtime (nearest filtering)

## Controls

- **Left click**: place building
- **Drag with left mouse**: place conduits (auto path)
- **Right click**: cancel selection or harvest
- **1?7**: building hotkeys
- **H**: harvest terrain
- **F**: convert forest to filter
- **R**: research view
- **M**: interplanetary view
- **F1**: toggle help
- **Esc**: menu

## Running the Game

```bash
cargo run
```

The default binary is `nanite_swarm`.

## Generating Art Assets

The art assets are generated procedurally from Rust code.

```bash
cargo run --bin build_graphics
```

Generated files:
- Terrain tiles: `assets/tiles/terrain_*.png`
- Building tiles: `assets/tiles/buildings/building_*.png`

## Project Layout (High Level)

- `src/main.rs` ? game loop + screen transitions
- `src/engine/` ? grid rules, pathfinding, drones, power
- `src/state/` ? simulation + save state
- `src/screens/` ? UI views and rendering
- `src/ui/` ? shared UI components and colors
- `graphics_gen/` ? procedural tile generator
- `assets/` ? generated art + JSON configs

## Data + Persistence

- Save file: `save.json` in project root (created via menu)
- Game data JSON: `assets/game_config.json`

## Known Gaps (Not Yet Implemented)

- Multi-planet persistence and resource transfer
- Advanced hazards (acid rain, freeze, heat)
- Deep balancing pass for economy and pacing

## Tech Stack

- **Language**: Rust 2021
- **Engine**: Macroquad
- **Assets**: Procedural PNG generation via `image` crate
- **Platform Targets**: Windows + WebGL (WASM)

## Tips for Development

- If textures appear blurry, confirm `FilterMode::Nearest` is applied
- Run `cargo run --bin build_graphics` after updating generator code
- The tile size in rendering is 28px; textures are 32px for padding/bleed
